## TMUX

I felt like I was getting a bit sloppy with TMUX, so I rewatched some tutorial videos to tighten up my workflows. The big things I'm going to be more intentional about:

- Using `(` and `)` to switch between sessions instead of reaching for the mouse
- Being more disciplined about placing terminals in the same pane rather than opening new ones unnecessarily

Small habits like these add up and I think they'll make me a lot faster day to day.

## USD.ai

I've been curious about USD.ai ever since I saw them speaking at Penn Blockchain. Today I spent time getting a solid understanding of their two token system. The design is interesting — there's a clear separation of concerns between the two tokens that I want to keep digging into.

## Clean Code

I felt like my code had been getting a little sloppy, so I went back and re-read parts of *Clean Code* by Bob Martin. Mostly refreshed myself on:

- **Naming conventions** — names should reveal intent, no mental mapping required
- **Function rules** — max parameters, keeping functions small

A lot of people push back on the strict function size and parameter count rules, but I find that following them forces you to create better structures and abstractions that really pay off later. It's less about the rules themselves and more about what they push you toward.
